✅ BACKEND LOGO FIX - ZUSAMMENFASSUNG
=====================================

## Problem
Das Logo wird im Filament Backend nicht angezeigt:
- ❌ Nicht in der Tabellen-Ansicht (ImageColumn)
- ❌ Nicht in der Formular-Vorschau (FileUpload)

## Ursache
Filament verwendet intern standardmäßig `asset()` für URL-Generierung, was für
Multi-Tenancy mit Subdomains nicht funktioniert.

asset() generiert: http://localhost:8000/storage/logos/...
Benötigt wird:     http://nknapijed.localhost:8000/storage/logos/...

## Lösung

### 1. FileUpload - Vorschau im Formular
Datei: app/Filament/Club/Resources/TemplateSettingResource.php

```php
FileUpload::make('logo')
    ->label('Logo')
    ->image()
    ->disk('public')
    ->visibility('public')
    ->directory('logos')
    ->imageEditor()
    ->maxSize(5120)
    ->getUploadedFileUrlUsing(function ($file) {
        if (is_string($file)) {
            return '/storage/' . $file;
        }
        return null;
    }),
```

### 2. ImageColumn - Anzeige in Tabelle
Datei: app/Filament/Club/Resources/TemplateSettingResource.php

```php
Tables\Columns\ImageColumn::make('logo')
    ->label('Logo')
    ->disk('public')
    ->visibility('public')
    ->getStateUsing(function ($record) {
        return $record->logo ? '/storage/' . $record->logo : null;
    }),
```

## Warum relative URLs?
- `/storage/logos/...` funktioniert automatisch für alle Tenant-Subdomains
- Keine Hardcodierung von Domain-Namen
- Browser ergänzt automatisch die aktuelle Domain

## Wenn Logo immer noch nicht erscheint:
1. Browser-Cache vollständig leeren (Strg+Shift+Del)
2. Browser DevTools öffnen (F12)
3. Network Tab öffnen
4. Seite neu laden
5. Nach "logo" oder "01K8KSS023SW8CYGG1QM6G5GZG.png" suchen
6. Status Code prüfen:
   - 200 = OK, Bild wird geladen
   - 404 = Route nicht gefunden
   - 403 = Zugriff verweigert
